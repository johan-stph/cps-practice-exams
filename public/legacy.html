e) Exact linearization

Retake 20212 cr

Given is с║Л = f(x, u), where x is the state and u is the input. Provide the most general system dynamics for which the linearization of f(x, u) in x = x* and u = u* is exact.

Solution

The first-order Taylor expansion is exact iff all higher-order derivatives vanish, i.e. f is affine in x and u:

f(x, u) = A x + B u + c  (constant matrices A, B and constant offset c).

f) LQR weighting matrices

Retake 20212 cr

Explain the difference in closed-loop system behavior between two LQR controllers C1 and C2, where the weighting matrices for the first are Q1 = 1001, R1 = 1, and for the second Q2 = 1001, R2 = 2.

Solution

Both weight the states equally, but C2 penalizes control effort twice as much (R2 = 2R1). Hence C2 uses smaller control inputs and the closed loop converges more slowly (less aggressively); C1 reacts faster at the price of larger actuation. Only the ratio between Q and R matters.

g) Timed automaton not simulatable by an FSA

Retake 20212 cr

Sketch a timed automaton with two states z1, z2 and one clock c1, which cannot be simulated by a finite state automaton.

Solution

The behavior depends on the real-valued clock c1: the full state is (z, c1) тИИ {z1, z2} ├Ч тДЭтЙе0, an uncountably infinite state space, and whether a transition may fire depends on the exact continuous clock value. A finite state automaton has finitely many states and no notion of time, so it cannot simulate this timing-restricted behavior (any automaton with a guard that actually constrains behavior works as an answer).

h) Reachability analysis for safety

Retake 20211 cr

Explain how reachability analysis of continuous-time dynamical systems verifies that no unsafe states are reached.

Solution

One computes the set of all states reachable from the initial set (for all allowed inputs/disturbances) over the considered time horizon тАФ in practice a guaranteed over-approximation of it. If this reachable set has an empty intersection with the unsafe set, no trajectory can reach an unsafe state, so the system is verified safe.

f) LQR тАФ scaling the state weight

g) Timed automaton тАФ clock regions

New2 cr

A timed automaton has a single clock c1, and the largest constant appearing in any clock constraint is 2. Enumerate all clock regions of the region abstraction. Why is this abstraction useful?

Solution

Regions: {0}, (0,1), {1}, (1,2), {2}, (2,тИЮ) тАФ 6 regions.

All clock valuations within one region satisfy the same clock constraints and lead to time-abstract equivalent behavior. Since there are only finitely many regions, verification problems (e.g. reachability) on the uncountable state space become decidable on the finite region automaton.

h) Reachability analysis тАФ inconclusive result

New1 cr

An over-approximative reachability analysis returns a reachable set that intersects the unsafe set. Can you conclude that the system is unsafe? What can be done next?

Solution

No. The intersection may be caused purely by the over-approximation error тАФ the result is inconclusive. Next steps: reduce the over-approximation (smaller time steps, tighter set representations, splitting the initial set) and re-run; or try to find a concrete counterexample trajectory (falsification) to prove unsafety.
